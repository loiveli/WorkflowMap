
Contains all the data about the information about the work required to complete the development of the game. At their core, all project management tools are just Excel in a trench coat, but there are some differences in how the tools present the data ^a75160

## Options

### [Jira](https://www.atlassian.com/software/jira)
Industry standard at AAA studios due to its scalability and enterprise focus. Relatively customizable and easy to use, but over time configuration changes may start to pile up creating a tangle of legacy workflows and fields that you cannot get rid of. Has plenty of add-ons to improve the user experience and feature set, but the base features should be enough for a smaller team

## [Notion](https://www.notion.com/)/[Linear](https://linear.app/)
I am grouping these two due to their simlar user experience and AI focus. Both are built to be highly configurable and the main benefit of these tools is to have your knowledge base and backlog in the same tool to remove friction. I have only used Notion, and while it does seem to be a good tool, their free tier seems quite limited.

## [Github/Gitlab projects](https://docs.github.com/en/issues/planning-and-tracking-with-projects/learning-about-projects/about-projects)
Both Github and Gitlab offer a project management tool, which brings the backlog closer to the version control and the code of the project. This is the easiest solution and at least the Github offering seems to be quite easy to configure.

## [Trello](https://trello.com/)
Probably the simplest option for project management out there, basically just a digital post it note board. Personally I find Trello too simple and not configurable enough, but for a small team it should work just fine
