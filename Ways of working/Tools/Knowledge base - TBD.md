Holds all the knowledge and documentation about the project, ways of working and everything that has been written down about the project.

## Options
### [Notion](https://www.notion.com/)/[Linear](https://linear.app/)
I am grouping these two due to their simlar user experience and AI focus. Both are built to be highly configurable and the main benefit of these tools is to have your knowledge base and backlog in the same tool to remove friction. I have only used Notion, and while it does seem to be a good tool, their free tier seems quite limited.

### [Obsidian](https://obsidian.md/)
Open source alternative to Notion/Linear. Obsidian is open source so you could host your own vault in Git to keep it synchronized between everyone but Obsidian also offers a simple built in sync solution with a 5$ subscription. Obsidian is highly customizable and as all the data is stored in markdown files it is easy to import/export files from/to obsidian

### [Confluence ](https://www.atlassian.com/software/confluence)
Industry standard for AAA studios, basically for the same reasons as JIRA and as both are Atlassian products they also integrate well. Not sure if I would recommend for a small team

### [Github Wiki](https://docs.github.com/en/communities/documenting-your-project-with-wikis/about-wikis)
By storing all the knowledge in your github repo in markdown you could utilize the Github wiki as your knowledge base, but to be honest at that point it would just make more sense to convert it to an obsidian vault and use all the features provided by obsidian.
